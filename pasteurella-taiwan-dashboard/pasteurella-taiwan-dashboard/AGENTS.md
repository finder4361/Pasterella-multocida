# AGENTS.md

## 專案目標

維護「臺灣多殺型巴斯德桿菌區域年度互動式儀表板」。本專案是純前端靜態網站，必須可部署到 GitHub Pages，且不需要後端伺服器。

## 不可破壞的資料定義

1. 「去重分離紀錄數」不是「分離率」。
2. 「菌株分布比例」＝某群組去重分離紀錄數 ÷ 篩選後全部去重分離紀錄數。
3. 「分離率」只有在同一統計單位同時存在 `陽性數` 與 `總檢體數／總送檢數` 時才能計算。
4. 若缺少分母，UI 必須明確顯示「無法計算分離率」，不得用菌株比例代替。
5. 去重優先使用 `去重鍵`；沒有時才用 `西元年 + 免疫室編號 + 分離部位`。
6. 排除 `納入年度統計 = 否` 與 `是否重複保管管 = 是`。

## 地區分類

- 核心地區：雲林縣＝雲林區；高雄市、屏東縣＝高雄屏東區。
- 非核心縣市依既有資料表 `病例區域` 與 `區域屬性` 顯示，不要在前端自行重新推測。
- 無法確認者保留 `待確認`。
- UI 必須能切換是否納入 `待確認`。

## 隱私

- 預設不得顯示畜主姓名。
- 新增地圖時只顯示縣市彙總，不顯示精確場址座標。
- 任何匯出功能預設排除畜主、電話、地址。
- 不得將使用者上傳的 Excel 自動傳送到第三方 API。

## 技術限制

- 保持 GitHub Pages 相容。
- 優先使用原生 JavaScript、HTML、CSS；若轉為 Vite/React，需保留自動部署工作流程。
- 不加入需要付費金鑰的服務。
- 外部 CDN 載入失敗時，要有可理解的錯誤提示。
- 所有介面使用繁體中文。
- 圖表需支援行動裝置與 PNG 下載。
- 任何新增功能都要更新 README。

## 驗收

每次修改後至少檢查：

1. `區域分類明細` 工作表可載入。
2. 民國年能正確換算西元年。
3. 重複保管管不會被重複計數。
4. 缺少分母時，分離率圖不會顯示錯誤數值。
5. 畜主欄預設隱藏。
6. GitHub Pages 路徑下資源使用相對網址。
