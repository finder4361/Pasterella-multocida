# 臺灣多殺型巴斯德桿菌區域年度互動式儀表板

這是一個可直接部署到 GitHub Pages 的靜態互動網站，支援：

- 本機上傳 Excel (`.xlsx/.xls`) 或 CSV
- 讀取公開 Google 試算表／發布到網路的 CSV
- 依西元年、病例區域、縣市篩選
- 去除重複保管管與重複分離紀錄
- 年度 × 區域圖、縣市排序圖、區域屬性構成、資料品質趨勢
- 下載圖表 PNG 與篩選後 CSV
- 預設隱藏畜主姓名

## 一、最快使用方式

1. 將整個專案上傳到新的 GitHub repository。
2. Repository → **Settings** → **Pages**。
3. Source 選擇 **GitHub Actions**。
4. 推送到 `main` 後，工作流程會自動部署。
5. 開啟網站，上傳：
   - `巴斯德資料菌株分析_區域年度整理.xlsx`
   - 優先讀取工作表：`區域分類明細`

## 二、建議 Excel 欄位

### 菌株明細模式

網站已針對下列欄位設置中文別名：

- `納入年度統計`
- `西元年` 或 `民國年`
- `免疫室編號`
- `分離部位`
- `去重鍵`
- `是否重複保管管`
- `原始縣市`
- `原始鄉鎮`
- `病例區域`
- `區域屬性`
- `資料品質註記`
- `畜主`（預設不公開顯示）

預設去重順序：

1. 排除 `納入年度統計 = 否`
2. 排除 `是否重複保管管 = 是`
3. 優先使用 `去重鍵`
4. 若沒有去重鍵，使用 `西元年 + 免疫室編號 + 分離部位`

### 真正分離率模式

若要計算真正「分離率」，資料中至少需要：

- `西元年`
- `病例區域` 或 `原始縣市`
- `陽性數`
- `總檢體數`／`總送檢數`／`病例總數`

公式：

```text
分離率 (%) = 陽性數 / 總檢體數 × 100
```

只有保菌株資料時，網站可呈現的是「分離株數」與「菌株分布比例」，不能宣稱為真正分離率。

## 三、Google 試算表連動

### 方法 A：發布到網路

1. Google 試算表 → **檔案** → **共用** → **發布到網路**
2. 選擇要發布的單一工作表
3. 格式選擇 CSV
4. 將網址貼到網站的「Google 試算表」欄位

### 方法 B：公開檢視網址

將檔案設為「知道連結的任何人皆可檢視」，然後貼上包含 `gid` 的網址。網站會嘗試轉換成 CSV 查詢網址。

> 若涉及畜主、電話、詳細地址或未公開研究資料，不建議直接公開原始試算表。可另外建立一個匿名化的「網站輸出」工作表。

## 四、隱私與研究品質

公開網站建議僅保留：

- 年份
- 縣市／病例區域
- 區域屬性
- 分離部位
- 彙總後菌株數或分離率

不建議公開：

- 畜主姓名
- 電話
- 詳細地址
- 可逆推出單一場戶的病例編號
- 未發表的 MIC、血清型或基因型明細

## 五、本機預覽

可直接雙擊 `index.html`。若瀏覽器限制本機 CORS，使用任一靜態伺服器：

```bash
python -m http.server 8080
```

再開啟：

```text
http://localhost:8080
```
